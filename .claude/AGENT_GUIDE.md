# Agent 사용 가이드

## 에이전트 구성

| 에이전트 | 역할 | 언제 사용 |
|----------|------|-----------|
| @planner | 설계 / 단계 정의 | 새 기능 추가 전 |
| @builder | 코드 구현 | 실제 개발 시 |
| @validator | 검증 / 테스트 | 구현 후 확인 |
| @analyst | KPI 분석 / 인사이트 | 보고서 작성 시 |

---

## 일반 작업 흐름

```
1. @planner → 설계 (무엇을, 어떻게)
2. @builder → 구현 (실제 코드 작성)
3. @validator → 검증 (Playwright 자동 확인)
```

---

## 에이전트별 상세 가이드

### @planner
**사용 시점**: 새 기능 추가 전, 설계가 불명확할 때

```
사용 예:
> @planner 포생산 공정 월별 MTBF 경보 시스템 추가하고 싶어

출력:
- 현상태 분석
- 구현 단계 제안
- 주의사항
```

---

### @builder
**사용 시점**: 명확한 요구사항이 있을 때

```
사용 예:
> @builder facility/page.tsx에 월별 수리 건수 트렌드 라인 차트 추가

필수 규칙:
- null 처리 준수
- /60 단위 변환 확인
- 기존 기능 유지
```

---

### @validator
**사용 시점**: 구현 완료 후, 데이터 변경 후

```
사용 예:
> @validator 홈 페이지 KPI 카드 전체 검증해줘

실행:
- Playwright MCP로 브라우저 자동화
- API 직접 호출로 데이터 검증
- 체크리스트 결과 보고
```

---

### @analyst
**사용 시점**: 임원 보고 준비, 데이터 해석 필요 시

```
사용 예:
> @analyst 2026년 1분기 설비 신뢰성 현황 분석해줘

출력:
- MTBF/MTTR 수준 평가
- 주요 이슈 식별
- 권고 사항
- 임원 보고용 요약
```

---

## 스킬 목록

| 스킬 | 역할 |
|------|------|
| kpi-analyzer | MTBF/MTTR 수준 판정 및 인사이트 |
| data-validator | 데이터 정합성 규칙 검증 |
| seed-runner | CSV → DB 재적재 |

---

## 핵심 데이터 규칙 (모든 에이전트 준수)

1. **stopCount = 0 → MTBF/MTTR = null** (절대 규칙)
2. **UI: null → "-"** (0 표시 금지)
3. **차트: null → 미표시** (0 막대 금지)
4. **단위**: 가동시간/정지시간은 분 → /60으로 시간 변환
5. **seed**: operatingTime·stopCount 모두 null인 행 → skip
